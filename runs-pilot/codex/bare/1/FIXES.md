# Dependency Vulnerability Fixes

- `axios` (`fixtures/webapp`): `npm audit` reported SSRF, credential leakage, denial-of-service, prototype pollution gadget, header injection, and request/response tampering advisories against `1.6.0` (including GHSA-8hc4-vh64-cxmj, GHSA-jr5f-v2jv-69x6, GHSA-4hjh-wcwx-xvwj, GHSA-3p68-rc4w-qgx5, GHSA-pjwm-pj3p-43mv, and related axios advisories). Updated to `1.18.1`, the non-major fixed version reported by npm, because it clears the vulnerable ranges.
- `express` (`fixtures/webapp`): `npm audit` reported XSS in `express` and vulnerable transitives through `body-parser`, `cookie`, `path-to-regexp`, `qs`, `send`, and `serve-static`. Updated `express` from `4.19.2` to `4.22.2`, the non-major fixed version reported by npm, so the Express dependency tree resolves to patched transitives.
- `body-parser` (`fixtures/webapp`, transitive): `npm audit` reported denial of service in versions below the patched range. Remediated by updating `express` to `4.22.2`, which no longer resolves the vulnerable `body-parser`.
- `cookie` (`fixtures/webapp`, transitive): `npm audit` reported invalid character handling in old `cookie` versions. Remediated by updating `express` to `4.22.2`, which resolves a patched `cookie`.
- `path-to-regexp` (`fixtures/webapp`, transitive): `npm audit` reported ReDoS advisories in old `path-to-regexp`. Remediated by updating `express` to `4.22.2`, which resolves a patched `path-to-regexp`.
- `qs` (`fixtures/webapp`, transitive): `npm audit` reported denial-of-service advisories in old `qs`; it was present through both `express` and `request`. Remediated by updating `express` to `4.22.2` and removing `request` through the importer migration.
- `send` (`fixtures/webapp`, transitive): `npm audit` reported template-injection XSS in old `send`. Remediated by updating `express` to `4.22.2`, which resolves a patched `send`.
- `serve-static` (`fixtures/webapp`, transitive): `npm audit` reported template-injection XSS in old `serve-static`. Remediated by updating `express` to `4.22.2`, which resolves a patched `serve-static`.
- `jsonwebtoken` (`fixtures/webapp`): `npm audit` reported key-type, key-retrieval, and default-algorithm verification advisories against `8.5.1` (GHSA-8cf7-32gw-wr33, GHSA-hjrf-2m68-5959, GHSA-qwph-4952-7xr6). Updated to `9.0.3`, the fixed major version reported by npm.
- `lodash` (`fixtures/webapp`, dev): `npm audit` reported command injection, ReDoS, code injection, and prototype pollution advisories against `4.17.20`. Updated to `4.18.1`, the fixed version reported by npm.
- `request` (`fixtures/webapp`): `npm audit` reported SSRF in `request` itself and marked no version-only fix available. Migrated `src/importer.js` from `request` to the already-used `axios` client and removed `request` from dependencies because there is no fixed `request` version.
- `form-data` (`fixtures/webapp`, transitive of `request`): `npm audit` reported unsafe boundary generation and CRLF injection, with no fix available through the `request` dependency path. Removed by migrating off `request`.
- `tough-cookie` (`fixtures/webapp`, transitive of `request`): `npm audit` reported prototype pollution, with no fix available through the `request` dependency path. Removed by migrating off `request`.
- `uuid` (`fixtures/webapp`, transitive of `request`): `npm audit` reported a missing buffer bounds check, with no fix available through the `request` dependency path. Removed by migrating off `request`.
- `pyjwt` (`fixtures/service`): `pip-audit` reported multiple advisories against `1.7.1`, including PYSEC-2022-202/GHSA-ffqj-6fqr-9h24 and newer fixed advisories requiring `2.13.0`. Updated `requirements.in` and `requirements.txt` to `pyjwt==2.13.0` and adjusted `app/auth.py` for the PyJWT 2.x `encode()` return type.
- `urllib3` (`fixtures/service`, transitive of `requests`): `pip-audit` reported redirect/header handling, decompression, and SSRF-related advisories against `1.26.5`, with the newest fixed advisory requiring `2.7.0`. Updated the compiled pin to `urllib3==2.7.0`, which is allowed by the existing `requests` constraint.
- `jackson-databind` (`fixtures/api-java`): OSV reported multiple denial-of-service and deserialization advisories against `2.13.0`. Updated to `2.22.0`, the latest Maven Central release, which remediates the older fixable advisories. OSV still reports GHSA-5jmj-h7xm-6q6v against `2.22.0`; the advisory names fixed releases `2.18.9`, `2.21.5`, and `2.22.1`, but none of those versions are available in Maven Central, so no version-only fix currently exists for that advisory.
- `commons-configuration2` (`fixtures/api-java`): OSV reported code injection and stack overflow advisories against `2.7` (including GHSA-xj57-8qj4-c4m6, GHSA-9w38-p64v-xpmv, and GHSA-xjp4-hw94-mvp5). Updated to `2.15.1`, which is beyond the fixed ranges.
- `commons-text` (`fixtures/api-java`, formerly transitive): OSV reported arbitrary code execution in `1.8` (GHSA-599f-7c49-w659). Added an explicit dependency on `1.15.0` so Maven resolves a patched version.
- `commons-lang3` (`fixtures/api-java`, formerly transitive): OSV reported uncontrolled recursion in `3.9` (GHSA-j288-q9x7-2f5v). Added an explicit dependency on `3.20.0` so Maven resolves a patched version.
